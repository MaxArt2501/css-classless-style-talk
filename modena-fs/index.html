Shadow DOM

The problem is that Shadow DOM's encapsulation is very strong. Not everyone is comfortable with it, and IMO it's not always the best choice. Ok for a UI Kit where you need visual consistency, not so much for pages, forms and widgets. Angular, with its new 80's look logo that fits so much this presentation, has always offered great flexibility for its encapsulation solutions, providing the Shadow DOM, emulated encapsulation (default one) and no encapsulation at all. The latter two allow us to use global stylesheets. Several others too: Svelte, Astro

?

With React… it depends. You can choose your own poison when it comes with styles. But, "natively", React comes with CSS Modules.

Nesting

 The cascade is part of the game, it's the first letter, it won't go away, so let it be the tool it's meant to be Why did it become a problem? Because more than a cascade, it became a flood. But now we can build the banks to contain it 
footer {
  display: flex;
}
footer button {
  color: ebony;
  background: ivory;
}
footer {
  display: flex;
  & button {
    color: ebony;
    background: ivory;
  }
}
There shoudn't be much difference, but nesting is so nice! And they worked on the spec so well we almost feels at home coming from SASS It's an expressive way to define a contextual style 120 117 17.2

 119 128 16.4 Not just CQs, but CSQs. Talked about them last year, so I'll be brief. CSQs could be used only with custom properties at the moment, but IMO they'll stay like that. Still useful, though!

Container style queries

 
@container style(--theme: dark) {
  p { ... }
}


@container style(--theme: rainbows) {
  header { ... }
}
We have always used custom properties to deliver single values: colors, spacings, sizes, etc. With CSQ, we can define a meaning - and not just a single value - and the corresponding style, with multiple rules. We could use a class, but classes are not inherited. 
@container style(--spacing: loose) {
  main { padding: 2rem; }
}

@container style(--spacing: compact) {
  main { padding: 0.5rem; }
}
We can respond to user preferences beyond what Media Queries can offer 
@container style(--orientation: vertical) {
  section { flex-direction: column; }
}

@container style(--orientation: auto)
  and (width > 10rem) {
  section {
    flex-direction: row;
  }
}
