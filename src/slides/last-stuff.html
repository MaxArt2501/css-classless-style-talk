Container style queries

 
@container style(--theme: dark) {
  p { ... }
}


@container style(--theme: rainbows) {
  header { ... }
}
We have always used custom properties to deliver single values: colors, spacings, sizes, etc. With CSQ, we can define a meaning - and not just a single value - and the corresponding style, with multiple rules. We could use a class, but classes are not inherited. 
@container style(--spacing: loose) {
  main { padding: 2rem; }
}

@container style(--spacing: compact) {
  main { padding: 0.5rem; }
}
We can respond to user preferences beyond what Media Queries can offer 
@container style(--orientation: vertical) {
  section { flex-direction: column; }
}

@container style(--orientation: auto)
  and (width > 10rem) {
  section {
    flex-direction: row;
  }
}
