But let's see how we can apply this pattern effectively.

Shadow DOM

The problem is that Shadow DOM's encapsulation is very strong. Not everyone is comfortable with it, and IMO it's not always the best choice. Ok for a UI Kit where you need visual consistency, not so much for pages, forms and widgets. Angular, with its new 80's look logo that fits so much this presentation, has always offered great flexibility for its encapsulation solutions, providing the Shadow DOM, emulated encapsulation (default one) and no encapsulation at all. The latter two allow us to use global stylesheets. Several others too: Svelte, Astro

?

With React… it depends. You can choose your own poison when it comes with styles. But, "natively", React comes with CSS Modules.

export default function Card() {

So, in the end, you only need to use one class, that we set to the component's name itself.

Many frameworks support CSS Modules, like Solid, Qwik…

It gets better.

 
<section>

  <style>
    @scope {
      :scope {
        color: navy;
        outline: 0.1em solid;
        padding: 1lh 1em;
      }
    }
  </style>
</section>
